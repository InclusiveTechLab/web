Examples of potential exclusions for Websites - Inclusive Tech Lab

Access account/profile menu

User opens account options.

Perceivable

  • Avatar unlabeled; blind user doesn't know it's clickable.
  • Avatar-only button unlabeled; screen reader announces "graphic."
  • Profile button uses avatar only; no text label for blind user.

Operable

  • Menu only opens on hover; not keyboard accessible.
  • Menu requires hover or gesture hold; not operable with keyboard.
  • Requires swipe hold gesture; inaccessible for motor impaired.

Understandable

  • Options phrased vaguely ("Your stuff"); user unsure of function.
  • Options listed in inconsistent order; confusing to memory-impaired users.
  • Options rearranged on each visit; confusing for memory impaired user.

Access support / help

User opens chat/help widget.

Perceivable

  • Chat widget unlabeled; screen reader can't find entry point.
  • Icon hidden behind overlapping elements; low-vision user doesn't find it.
  • Widget hidden behind overlapping footer; low vision user cannot see.

Operable

  • Widget anchored in corner; switch user can't reach.
  • Chat widget floats and blocks page content; screen reader user can't reach background.
  • Requires multi finger gesture to activate; inaccessible for prosthetic hand.

Understandable

  • Bot replies unclear or too complex; user with cognitive disability can't parse.
  • Bot uses complex multi-turn dialogue; confusing for autistic users.
  • Bot responds with long paragraphs; overwhelming for user with ADHD.

Adjust presentation

User zooms, resizes, or toggles mode.

Perceivable

  • Text in images doesn't scale; low-vision user loses info.
  • Zooming crops content; low-vision user loses information.
  • Content overlaps after zoom; low vision user misses context.

Operable

  • Resize breaks layout; keyboard navigation impossible.
  • Theme toggle placed at screen bottom; unreachable for screen magnifier users.
  • Requires rotating device orientation; inaccessible for fixed mount devices.

Understandable

  • Layout shifts unpredictably; neurodiverse user disoriented.
  • Layout reflows unpredictably, disorienting ADHD user.
  • Terminology for modes unexplained; confusing for dyslexic users.

Authenticate/login

User enters credentials, uses autofill, or 2FA.

Perceivable

  • CAPTCHA without audio/text alternative excludes blind users.
  • Autofill suggestions not spoken by screen readers.
  • Login feedback uses subtle color change only; colorblind user excluded.

Operable

  • Requires SMS code under time pressure; motor-impaired user excluded.
  • 2FA code requires switching apps quickly; inaccessible for memory-impaired users.
  • Requires scanning QR code; impossible for blind users.

Understandable

  • Error "invalid login" too vague; dyslexic user doesn't know which field is wrong.
  • Password rules hidden until after failure; user with dyslexia repeatedly locked out.
  • 2FA code delivered in complex sentence; confusing for dyslexic user.

Browse sequences

User swipes, clicks arrows, or presses keys in a carousel.

Perceivable

  • Carousel images lack alt text; blind users can't perceive items.
  • Arrows blend into background; low-vision users can't perceive controls.
  • Carousel dots lack labels; blind user cannot know current position.

Operable

  • Carousel auto-rotates with no pause button; motor/vestibular issues triggered.
  • Auto-play requires fast reactions to pause; inaccessible for slow motor users.
  • Swipe speed must be exact; inaccessible for motor impaired users.

Understandable

  • No progress indicator; user doesn't know how many slides exist.
  • Carousel resets to first slide when reopened; memory-impaired user loses place.
  • Carousel restarts unexpectedly; confusing for users with short term memory challenges.

Consent/permissions

User responds to cookie/notification prompts.

Perceivable

  • Popup not announced; blind user unaware.
  • Pop-up placed at screen bottom, hidden from magnifier users.
  • Consent prompt uses transparent overlay; invisible to low vision users.

Operable

  • Close button too small; motor-impaired user can't dismiss.
  • Requires precise tap on "X" to dismiss; inaccessible to motor-impaired users.
  • Requires precise voice phrase "Allow notifications"; excludes speech diversity.

Understandable

  • Language in banner overly legalistic; user with cognitive disability can't consent meaningfully.
  • Legal language overly dense; user with cognitive disability clicks without comprehension.
  • Consent terms vague; user cannot make informed decision.

Perceivable

Operable

  • Share button requires long-press gesture; inaccessible to prosthetic users.
  • Share requires drag motion to dock; inaccessible for users with tremors.

Understandable

  • Share menu overloaded; user with ADHD overwhelmed.
  • Share menu full of irrelevant networks; confusing for novice users.
  • Share destinations unlabeled; confusing for novice users.

  • Expansion requires horizontal swipe; inaccessible to trackball users.
  • Requires double click with precision; tremor users excluded.

Understandable

  • Expansion changes layout unexpectedly; autistic user loses context.
  • Section collapses unexpectedly when tabbing, disorienting autistic users.
  • Section labels inconsistent; user cannot predict what will appear.

Filter/refine results

User applies categories, sliders, or checkboxes.

Perceivable

  • Slider has no text equivalent; blind user can't perceive range.
  • Filters displayed only as icons (e.g., funnel); low-literacy user can't interpret.
  • Filters hidden behind color icons; blind users unaware.

Operable

  • Slider requires precise dragging; motor-impaired user can't operate.
  • Multi-select requires holding Shift/Ctrl; inaccessible for one-handed users.
  • Multi column filter requires simultaneous scroll and click; inaccessible for one handed users.

Understandable

  • Filter labels unclear; ADHD user misinterprets categories.
  • Filter labels use insider terminology (SKUs") confusing laypeople."
  • Filter categories overlap semantically; confusing for cognitive disability.

Follow/subscribe

Perceivable

  • Bell icon unlabeled; blind user doesn't know purpose.
  • Bell notification only shows by color change; colorblind user excluded.
  • Subscription confirmed only with subtle vibration; DeafBlind users excluded.

Operable

  • Tiny icons hard to target; motor-impaired user struggles.
  • Requires double-tap on small icon; inaccessible for tremor users.
  • Requires gesture double tap on small icon; inaccessible for tremor users.

Understandable

  • No confirmation; user with memory issues unsure if subscription worked.
  • No clear explanation of what "Follow" entails (emails? notifications?); user hesitant.
  • Label "subscribe" conflated with payments; confusing for novice users.

Perceivable

  • Hover details appear outside magnification lens; low vision user misses info.

Operable

  • Touch users cannot replicate hover behavior.
  • Requires hover dwell time; inaccessible for users with fast tremors.

Understandable

  • Tooltip disappears too quickly; dyslexic user can't finish reading.
  • Tooltip uses unexplained abbreviations; cognitive load too high.
  • Tooltip uses icons with no text; incomprehensible for neurodiverse user.

Interact with media

User plays or pauses media.

Perceivable

  • Captions auto-hide after seconds; Deaf user loses content.
  • Media controls low contrast in dark mode; low vision user excluded.

Operable

  • Autoplay video without pause; keyboard users can't stop it.
  • Media controls require fine motor input; inaccessible for tremor.
  • Requires swipe gesture to scrub; inaccessible for keyboard only.

Understandable

  • Transcript missing; Deaf user can't understand dialogue.
  • Audio description uses technical jargon; confusing for new users.
  • Captions filled with non literal idioms; confusing for non native speakers.

Jump within page

User selects "skip link" or "back to top."

Perceivable

  • Skip link hidden from screen readers.
  • Back to top only appears visually when scrolling not announced to Assistive Technology.
  • Skip link styled as 1 pixel transparent element; invisible to low vision user.

Operable

  • Control only appears on mouse hover; not available via keyboard.
  • Requires rapid double-tap gesture on mobile; impossible for tremor.
  • Requires voice command phrasing exactly; excludes users with speech differences.

Understandable

  • Skip link text unclear ("jump" instead of "skip to content"); user unsure of purpose.
  • Skip link labeled with technical jargon ("anchor link") not meaningful to general users.
  • Back to top label not meaningful; some users expect scroll not jump.

Make a choice from options

User picks from dropdowns, radios, or checkboxes.

Perceivable

  • Custom dropdown not exposed to screen reader.
  • Options presented only visually (flag icons); blind users miss them.
  • Options presented only visually (flag icons); blind users miss them.

Operable

  • Dropdown only opens with mouse; can't be tabbed to.
  • Multi-select requires rapid tapping; inaccessible for switch users.
  • Requires long press to open; inaccessible for motor impairments.

Understandable

  • Options poorly grouped; user can't predict results.
  • Options overlap conceptually (e.g., newsletter" vs. "updates"); user confused."
  • Option labels too similar; user with cognitive disability cannot distinguish.

Navigate with links

User clicks/taps a link in breadcrumbs, footer, or trust section.

Perceivable

  • Link text says "Click here"; blind user doesn't know destination.
  • Link color indistinguishable from body text for colorblind users.
  • Link text presented only in uppercase; harder for screen reader pronunciation.

Operable

  • Small links too close together; motor-impaired user cannot reliably select.
  • Links open in new tab without warning, confusing keyboard/screen reader users.
  • Links spread across multiple lines; hard for motor impaired user to select.

Understandable

  • Breadcrumb order inconsistent; dyslexic user cannot follow hierarchy.
  • Breadcrumb order presented right-to-left, confusing for cultures with left-to-right reading.
  • Link destinations unclear; user with autism becomes anxious about outcomes.

Open navigation menus

User taps/clicks a menu icon, presses Enter, or hovers to expand options.

Perceivable

  • Menu icon unlabeled; screen reader user cannot perceive it.
  • Menu opens with animation too fast for users with low vision to track.
  • Menu opens with low contrast animation; user with color vision deficiency cannot perceive change.

Operable

  • Menu only opens on hover; keyboard user cannot activate it.
  • Gesture-only swipe to open menu excludes switch users.
  • Requires pinch gesture to open; inaccessible for users with limited hand mobility.

Understandable

  • Menu structure changes between pages; user with cognitive disabilities confused.
  • Icon uses metaphor (hamburger) that some users don't understand.
  • Menu items reorganize unpredictably; user with ADHD cannot maintain orientation.

Perceivable

Operable

Understandable

  • Error instructions unclear; dyslexic user doesn't know correction needed.
  • Input requires specific format with no examples; dyslexic user guesses incorrectly.
  • Input rejects unexpected but valid formats; frustrating for dyslexic users.

Understandable

Scroll vertically/horizontally

Perceivable

  • Scrollbar hidden by design choice; magnifier user cannot perceive boundaries.

Operable

  • Horizontal scroll traps keyboard focus; user can't reach content.
  • Scroll requires two-finger gesture on laptop touchpad; inaccessible to users missing fingers.
  • Requires fine two finger gesture; inaccessible for users with limited dexterity.

Understandable

  • Content reflows unpredictably; user with cognitive challenges loses place.
  • Continuous scroll makes it unclear where sections begin or end.
  • Endless scroll disrupts logical reading order; confusing for screen reader users.

Understandable
